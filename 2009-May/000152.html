>
> - Instead of getpagesize(), can we use a static _getpagesize() that
> looks for PAGE_SIZE being defined on the command line? If defined, it
> just uses that value. That should be agreeably inlined by gcc and saves
> numerous jumps and calls to sysconf, since we can get the value via
> autoconf (while obeying what the user thinks it should be).

The problems with previous cryopid version is there, PAGE_SIZE. Latest
libc, kernel headers changed that. I'm ok with static _getpagesize() to
avoid too much calls to sysconf, but I don't think define PAGE_SIZE on the
command line is ok because PAGE_SIZE is system depending.
What about a global variable that store getpagesize() value? or something
similar with macros?
