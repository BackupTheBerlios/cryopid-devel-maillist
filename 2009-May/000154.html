On Wed, 2009-05-27 at 11:14 +0200, Michele Alberti wrote:

> The problems with previous cryopid version is there, PAGE_SIZE. Latest
> libc, kernel headers changed that. I'm ok with static _getpagesize() to
> avoid too much calls to sysconf, but I don't think define PAGE_SIZE on the
> command line is ok because PAGE_SIZE is system depending.
> What about a global variable that store getpagesize() value? or something
> similar with macros?

Well, the idea accomplished two things:

1 - Eliminate so many calls to sysconf()
2 - Allow for ease of cross compiling for systems that have huge pages

Now, this is not an immediate concern, however it would be good if
changes now allow for it.

So if _getpagesize() obeyed what the user wants, when cross compiling,
it means distro packagers have an easier life and need to manage less
patches to support their various platforms.


> It's ok for me. I don't know if people still uses 2.4 version.

Many appliances do.. but I don't know how many of them actually use
cryopid for checkpointing. However, keeping the support means some of
them may start using it. I'm in the air regarding this.
