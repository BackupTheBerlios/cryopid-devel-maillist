Hi all,
this is a call for ideas.
The problem is explained here:

https://lists.berlios.de/pipermail/cryopid-devel/2009-March/000246.html

In particular, these are my thoughts:

Cryopid store all information needed to restore the process later.
So probrably even information about the process state, PID in particular
(somewhere in vma?). The result is that when restoring the process, the
getpid syscall wrapper of glibc return the old PID even if the process has
got a new one. So many things can't work, like reise() syscall.

I need ideas, INFO about Thread Local Storage (I think that it's the
origin of the problem, is it right?) and it's implementation in glibc.
Please help me, join #cryopid on IRC or just post here in the ml.
