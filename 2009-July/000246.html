	Hello,

Michele Alberti wrote:
 > Now the problem is how to find that getpid location.
 > I thought to search a signature of machine code but then I realized
 > that it's not a portable method.
	
	As it is libc exported symbol, you could just the dlsym() function :-)

	And I think the most 'clean' method would be to create own 'minimal' 
iplementation of this function, which will work always (something like
	mov    $0x14,%eax
	int    $0x80
	ret
) and just paste resulting machine code (actually 8 bytes - 
0xb8,0x14,0x00,0x00,0x00,0xcd,0x80,0xc3) over the libc getpid() function 
address when writing the image. This way, the resulting binary will 
always use the getpid kernel syscal and no caching will ever happen.
	
	Just my 2 cents :-)
