No. VDSO (virtual dynamic shared object) it's something that created by kernel and used to implement syscalls with more efficient way then 'int 0x80'
This VDSO considered by dynamic linker (ld-linux.so) as any other 
shared library and it's address is saved into frozen process images.
Unfortunately for CryoPID (and statifier for that matter), when frozen process is run again VDSO address will be different - address space randomization.
Obviously nothing good can came from it. (Unless segfault count as  good :)

On the other hand mmap_min_addr is just security measure - process not allowed to mmap memory below address in this file.
(OK, may be root process is allowed - I don't remember)
