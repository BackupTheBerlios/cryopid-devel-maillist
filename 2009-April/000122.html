Hi, guys.

I was able to build CryoPID on Fedora Core1 i386 and after a bit more changes even use it !

Attached is a patch. 
But be careful, I don't think it should be applied as is, thought.

Most changes are trivial (like moving sys/ptrace.h) above linux/user.h) and I hope harmless. Another ones are less harmless
and should be subject to autoconf magic:
 - removed -fno-stack-protector option
 - added define user_desc modify_ldt_ldt_s - it was a previous name of user_desc

And the one I have most doubt about is changing RESUMER_END from
0x00800000 to 0x8000
Obviously this change doesn't need for compilation to succeed.
But without it when I was trying to freeze program I got message:
" Ignoring map - looks like resumer" from the cp_w_vma.c
And indeed a number of libraries on my system were mapped into this area.

Regards,
Valery.
