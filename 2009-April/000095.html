Some others on the list (not sure if you're still subscribing, so making
sure you are CC'ed) have taken interest in reviving the project. At the
least, we'd like to fix the outstanding build issue bugs (due to kernel
shifts) which resulted in cryopid being dropped from Debian.

In order, we'd like to:

  * Fix the bugs noted above
  * Work on advancing the roadmap

Cryopid is not the easiest thing to wrap ones mind around, though some
of us have some kernel experience. Our biggest goal is to get it back
into distro circulation while showing evidence of active development. We
hope this will bring more developers to the table who can really help
advance the roadmap.

Is a fork OK with you? If so, do you have some kind of repository that
allows us to import and preserve the history to date?
