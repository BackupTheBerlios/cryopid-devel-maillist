Hi guys,
I build cryopid for my linux box and tried to test some tester (example)
code.
I tried sigtest.c, and freezing goes well. But when I tried to resume the
process, it encounter problems. It is blocked!
So I read sigtest.c code and this in particular:

for(;;) {
        raise(SIGUSR1);
        sleep(2);
        raise(SIGUSR2);
        sleep(2);
}

The problem is that after resume the process, it has got a new PID and so
the raise syscall failed. In fact if I run this command:

$ kill -SIGUSR1 <new PID>

all goes well, the process caught the right signal.
If you put a getpid() inside for(;;) statement, you will notice that
getpid() returns the old pid, the pid before freezing the process.

How can I deal with it? How can I modify the code to obtain the getpid()
returns the right pid?
