Based on our discussion via IRC, I have added the following wiki page:

http://sharesource.org/project/cryopid/wiki/Roadmap/

Please feel free to add to it if I've missed anything. This covers the
next immediate release (0.5.9-2) which just puts support for linux-2.4
in its own branch. At that point, someone can back port interesting
things from the 2.6 tree. The 2.4 branch will always stay at
0.5.9-something, but can release independently of the 2.6 tree.

After we do what we discussed, I think we can tag in 0.6

The only thing I added that we did not discuss was the possibility of
creating a cryopid-bug tool, which could send us the information that we
want. One of the difficulties with cryopid for users is knowing how to
report bugs with useful info and where.

What I did not list was fixing x86_64, however if you want to take that
task, feel free to put it on the roadmap. If you need access to edit the
wiki, just e-mail me your ShareSource user name.

Cheers,
