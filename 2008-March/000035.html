You should have it. Its put into place via a kernel `make
headers_install', so kernel-headers should have installed it.

The file asm/prctl.h is just a sanity checker, see here:

#ifndef __ASM_STUB_PRCTL_H
#define __ASM_STUB_PRCTL_H
# if defined __x86_64__
#  include <asm-x86_64/prctl.h>
# elif defined __i386__
#  error asm-i386/prctl.h does not exist in the i386 architecture
# else
#  warning This machine appears to be neither x86_64 nor i386.
# endif
#endif /* __ASM_STUB_PRCTL_H */

If asm-x86_64/prctl.h exists, you should just be able to include it
directly in lieu of asm/prctl.h (in stub.c) It could be that the headers
package just left it out for some reason.

If asm-x86_64/prctl.h does not exist, something is wrong with the
kernel-headers package.

Someone on fedora-users may be able to give you better advice, as I said
its been a while for me.

Hope this helps :)
