On Wed, 2009-02-25 at 23:25 +0530, Rohit Banga wrote:
> hello
> i haven't really studied the source code in detail. just noticed an
> error in fork2.c
> 
> switch(helper_pid = fork()) {
> 
> case 0: /* parent */
> 
>                 waitpid code 
> 
> default:  /* child */
>                 execve code
> }
> 
> 
> fork returns 0 in child and not in parent ?

It makes a lot more sense when you keep in mind that bot child and
parent are going to run through the cases at the same time. One will be
a parent waiting for the result of the child, one will be the child
actually talking to execve.

The child will be long gone before the parent gets out of the switch().

All of this, of course, depending on fork() working in the first place.
